         of the repeaters in `CHIRP Lists/local WNC ham.csv`.

WHY THIS DOC EXISTS
-------------------
`WCARS Repeater List.txt` lists only "Tone In" — the CTCSS tone required to ACCESS
(key) the repeater (the UPLINK tone). It does NOT say whether the repeater transmits
a tone on its OUTPUT (the DOWNLINK). TSQL only works if the repeater transmits a
matching tone on its output; otherwise TSQL mutes the receiver and you miss the
repeater entirely. RepeaterBook's detail page lists Uplink Tone and Downlink Tone
SEPARATELY, so it can answer the output-tone question. This doc records that check.

DECISION RULE (owner's standard)
--------------------------------
- Downlink tone documented AND equals uplink  -> TSQL (source-ID + won't miss).
- Only an uplink/access tone documented (downlink blank / CSQ / mismatched)
                                              -> Tone (encode-only): key with the
     access tone, carrier RX so the repeater is NEVER missed.
- No tone documented at all                   -> carrier.
Note: ham repeater outputs are each on a UNIQUE frequency, so carrier RX has no
source-confusion downside (that only matters on shared channels like GMRS outputs).
